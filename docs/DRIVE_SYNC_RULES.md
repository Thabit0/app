# قواعد المزامنة عبر Drive/OneDrive بدون API

البرنامج لا يستخدم Google Drive API ولا OneDrive API. هو يتعامل مع مجلد محلي عادي، وبرنامج المزامنة الخارجي ينسخ الملفات بين الجهازين.

## قاعدة مهمة

لا يقرأ جهاز المحل أي job إلا بعد ظهور `READY.flag`.

هذا يمنع مشكلة وصول `post.json` قبل الصورة.

## ترتيب إنشاء job

1. إنشاء مجلد job.
2. حذف `READY.flag` إن وجد.
3. نسخ الصورة.
4. كتابة الكابشنات.
5. كتابة `post.json`.
6. كتابة `READY.flag` آخر شيء.

## عند التنفيذ

1. جهاز المحل ينشئ lock.
2. يجهز خطة التنفيذ في `plans`.
3. يفتح المنصات أو يجهز المهمة حسب الإعداد.
4. يكتب status.
5. يكتب DONE/FAILED/NEEDS_REVIEW.

## إذا كان جهاز المحل طافي

إذا اشتغل لاحقًا ووجد منشورات متأخرة:

- أقل من 6 ساعات: يعالجها.
- أكثر من 6 ساعات: يضعها `NEEDS_REVIEW` افتراضيًا.
