# Dawish Content Studio v2.0 — First Build

برنامج Windows واحد لجهاز المالك وجهاز المحل، مع مزامنة Cloudflare.

## ما تم تنفيذه

- ربط الجهاز أول مرة فقط: Worker URL + ADMIN_TOKEN أو SHOP_TOKEN.
- حفظ الـ Token محميًا بواسطة Windows DPAPI للمستخدم الحالي.
- شاشة رمز دخول إلزامية قبل ظهور البرنامج.
- رمز المالك: 200588.
- رمز جهاز المحل: 000000.
- واجهة المالك: استيراد مجلد كامل، كل صورة منشور مستقل، تحديد جماعي، جدولة جماعية، توزيع تلقائي، وفحص قبل الرفع.
- النص الثابت: `زوروا متجرنا بالبايو لرؤية باقي المنتجات` مع الهاشتاقات المعتمدة.
- Instagram: الموقع الإجباري الافتراضي `مدينة الزلفي`.
- TikTok: الموقع الإجباري الافتراضي `عطارة الدويش`، صورة فقط، بدون أغنية أو صوت.
- Snapchat: الصورة الأصلية فقط، بدون موقع أو كتابة أو هاشتاقات.
- واجهة جهاز المحل: فتح تسجيل الدخول للمنصات في المتصفح المحلي، تشغيل المحطة، Heartbeat، وعرض المنشورات المستحقة.
- GitHub Actions للبناء وإخراج برنامج Windows ذاتي الاحتواء.

## ملاحظة تسجيل المنصات

أزرار الحسابات تفتح المواقع الرسمية في متصفح Windows. المتصفح يحتفظ بالجلسات محليًا على جهاز المحل. لا تُرفع كلمات المرور أو الجلسات إلى Cloudflare.

## الرفع إلى GitHub

ارفع محتويات هذا المجلد إلى جذر المستودع، ويجب أن تظهر مباشرة:

- `.github`
- `cloudflare`
- `src`
- `DawishContentStudio.sln`

ثم افتح Actions وشغّل `Dawish v2 Windows Release`، أو انتظر Workflow التحقق عند Push.

## Cloudflare

استخدم `cloudflare/worker.js` في Worker الحالي، مع Bindings:

- D1: `DB`
- R2: `MEDIA`

ومع Secrets:

- `ADMIN_TOKEN`
- `SHOP_TOKEN`
